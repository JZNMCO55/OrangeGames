# Phase 0 — 设计 pillars + spine 决策

> 状态：**设计共识，未进入实现**。本文件是首游的「宪法」——之后每个机制 / 关卡 / 功能都拿这 3 条 pillar 当过滤器：都不服务 → 砍。
> 记录日期：2026-05-26。归属：OrangeGames 首游（Ori-like 平台跳跃，流体/史莱姆主角）。
> 配套执行规格见 [`spike-01-blob-feel.md`](spike-01-blob-feel.md)。

---

## 什么是 pillars，为什么它排在所有工作前面

设计 pillars = 2-4 句短到能背下来的话，捕捉这游戏的核心承诺。它是宪法：每加一个机制、设计一关、砍一个功能，都拿它量。

对本项目尤其关键，因为开发者最大的工程风险是**无限打磨、永不开始 / scope 失控**——pillars 是唯一能告诉你"什么时候够了、该砍了"的尺子。这里写成**可证伪**版（而非空泛口号），让 spike 能直接验证最危险的那条。

---

## Spine 决策（2026-05-26 用户拍板）

两条核心轴里，**形态切换 / 战斗 = 脊柱**；context-sensitive 液态穿行 = 辅助层（连接战斗的移动/探索层）。

> ⚠️ **诚实标记**：这是两条路里**内容风险更高**的一条。"成为你打败的东西"天然让玩家预期一个 roster，而每个形态 = moveset + 动画 + 美术 + 平衡 = 内容长杆，是单人首作头号失控源。下面的 containment（限时回落 + 形态一鱼两吃）就是为了让这条脊柱对单人可生存而焊死的纪律。

（备选轴"液态穿行为脊柱"被否：它内容更可控、更对口渲染/sim 长板，是助手的推荐，但用户更想要变身能力幻想。决策归用户所有。）

## 3 条 Pillars（可证伪）

### P1 ——「击败即获得一种独立能力」
打败持有形态的敌人 → 你**真的变成它**，拿到它完整的行为模组与招牌能力（不是 +数值）。变身后就是那个形态，流体退场。

> **证伪**：拿到新形态后操作 / 玩法没本质变化（只是换皮）→ 这条垮。新形态必须改变玩家**怎么按键、怎么走位**。

### P2 ——「能力是创造性工具，横跨战斗与解谜」
每个形态的招牌能力不止用于战斗——在场景 / 解谜里也有独特优势，**玩家自己发现"这能力还能这么用"**。能力越互异，想象空间越大。这也是内容**净赢**：每个形态一鱼两吃（战斗 + 解谜），单位内容榨出更多玩法 → 用更少的形态撑起更丰富体验，强化"少而深"。

> **证伪**：某形态能力只在战斗有用、世界里没有第二种用途 → 只值半个形态；若解谜只有唯一指定解法、没有"没想到还能这样"的余地 → 涌现不足，这条垮。

### P3 ——「流体是本体，形态限时回落」
流体是你的真身与归宿：承担**所有穿行**（context-sensitive：穿墙隙 / 水中 / 平台各异）+ 游戏视觉招牌。形态是借来的、**有时间限制**的状态，超时自动变回流体。时间限制三重作用：

1. **逼你果断、有意识地用形态**（不能赖着不走）——这就是"切换是动词，不是菜单"的落点。
2. **把穿行复杂度永远锁在唯一的流体本体上**（内容 containment）——形态数量翻倍，穿行调校不翻倍。
3. **"变回流体"成为节奏锚点**。

> **证伪（生产层）**：若某形态还需要自己**整套穿行 / 平台手感**调校 → 太贵，砍。形态只背战斗 + 招牌能力 moveset，走路永远还给流体。

## 内容 containment（焊死的 scope 纪律）

脊柱选了高内容风险的一条，护栏靠两条机制把每个形态的**表面积**压住：

1. **限时回落**：形态有时间上限，超时强制变回史莱姆（Mega Man 武器能量 / Kirby 受击掉能力那种）。穿行复杂度集中在唯一的流体本体。
2. **一鱼两吃**：每个形态的招牌能力同时是战斗工具和解谜 verb，不为单一用途造内容。

**硬指标（垂直切片）**：中立流体 + **3 个战斗形态封顶**。任何形态若不能用「战斗 moveset + 1 招牌能力 + 剪影 + shader」做出来、还要求自己一整套穿行手感 → 推迟或砍。

---

## 设计风险登记（撞到时再解，先记下）

- **timed-puzzle 陷阱**：「限时形态 + 解谜」容易滑成限时解谜，玩家普遍讨厌"还在想就变回去了"的挫败。处理方向（待 Spike 2 验证）：解谜做成**决断型**（看准→变身→果断一击）而非慢思考型；或进解谜情境时计时放宽 / 暂停。**别做成边想边倒计时。**
- **ability-gated 关卡复杂度**：形态能力用于解谜 = Metroidvania 式能力门控，关卡设计须追踪"此处玩家能拿到哪些形态"。属工程 / 关卡规划阶段（暂不动），记此备忘。

---

## 对 spike 路线的影响

- **Spike 1（史莱姆 base 手感）地位更重**：它现在是**所有形态共用的穿行地基**，不是某一个形态的事。优先级不变，先做。详见 [`spike-01-blob-feel.md`](spike-01-blob-feel.md)。
- **Spike 2 升级为生死 spike**：脊柱是形态切换，最危险的假设就是"切换 + 能力 + 限时回落好不好玩 + 形态能不能保持便宜"。垂直切片必须证明 **P1 + P2**。范围：base ↔ 1 个形态 + 一个招牌能力 + 计时回落（计时机制原型很便宜，顺手验）。

---

## 这不是什么（防误读）

- pillars **不是**功能清单——是过滤器。"做不做某功能"先问"服务哪条 pillar"。
- spine 选了形态切换，**不代表** context-sensitive 液态被放弃——它降级为辅助的穿行/探索层，仍是流体本体的招牌（P3）。
- 形态 = **临时 + 全用途**（战斗 + 解谜均可），靠时间限制 containment；**不是**永久全能角色（那等于"形态数 × 一整个游戏的角色工作量"，已否决）。
